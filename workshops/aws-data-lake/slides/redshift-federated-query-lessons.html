IAM Role — 3 Permissions Needed

  • Secrets Manager — read federated creds
  • Glue Data Catalog — Spectrum metadata
  • S3 GetObject / ListBucket — data reads

Attach to Redshift Namespace, not Workgroup.

Federated Query — How It Works

  • Creates external schema FROM POSTGRES
  • Secret stores read-only Postgres credentials
  • Live query — Aurora bears the read load
  • Read-only; no writes to Aurora possible

Spectrum — How It Works

  • Creates external schema FROM DATA CATALOG
  • Metadata lives in Glue (even without Glue console)
  • DDL defines external table over S3 prefix
  • Billed per TB scanned — partition to cut cost

Hard-Won Lessons

8 Gotchas That Will Break Your Setup

⚠ These cost the most time in the original setup
1

SCRAM vs md5 — The Final Blocker

Fix: run SET password_encryption='md5' and CREATE USER in the same session — Aurora defaults to scram-sha-256, which Redshift can't speak, and a new connection reverts the setting.

2

Never Use the rds!cluster-… Master Secret

Fix: create a dedicated redshift_fed md5 user with its own secret — the master secret auto-rotates back to SCRAM, re-breaking the connection every rotation.

3

Glue Is Required Even Without the Glue Console

Fix: grant the IAM role glue:CreateDatabase — CREATE EXTERNAL DATABASE writes to the Glue Data Catalog silently and schema creation fails without it.

4

Test the Network From Inside the Redshift VPC

Fix: use openssl s_client from inside the VPC — a passing psql from your laptop takes a different network path and proves nothing about what Redshift itself can reach.

5

TLS Fixes Happen on Aurora — Not Redshift

Fix: resolve cert/TLS issues on the Aurora side (CA identifier, SG rules, VPC routing) — CREATE EXTERNAL SCHEMA … FROM POSTGRES has no SSL knob of its own.

6

Keep Everything in the Same Region

Fix: keep Redshift, Aurora, and the S3 data all in the same region — cross-region adds latency and egress charges (the TICKIT sample defaults to us-east-1, so copy it first).

7

LOCATION Is a Folder Prefix, Not a File

Fix: always end LOCATION with / — a file path instead of a folder prefix silently returns 0 rows with no error.

8

Partitioned Tables Need Explicit ADD PARTITION

Fix: register every partition with ALTER TABLE … ADD PARTITION — Spectrum doesn't auto-discover partitions and there's no crawler for it.
